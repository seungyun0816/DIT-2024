웹에서는 예측 불가능한 숫자의 다수의 이용자가 언제든지 접속 가능하기 때문에
사용자들의 데이터를 전부 가지고있기에는 무리가 있기 때문에 데이터를 가지지 않게 만들어졌다.
-------------------------------
JSP

JAVA기반 웹 프로그래밍 언어
JSP는 실행시 Servlet으로 변환되며, 변환되지 않으면 브라우저는 이를 해석 할 수 없음.

화면 출력 및 데이터의 처리를 간편하게 하기 위해 만들어짐.

기본적인 형태는 HTML을 띄고있으며
JSP코드가 필요한 곳에 <% ~%>형식으로 삽입됨
-------------------------------
JSP구성 요소

스크립트 | 스크립트는 페이지내에 동적 데이터를 생성하고 이는 3가지로 분류된다
  -스크립트릿(<% ~ %> : JSP페이지 내부에 JAVA소스코드를 삽입
  -표현식(<%= ~ %? : HTML요소 내에 값(Value)을 넣기 위해 사용
  -선언(<%! ~ %> : JSP 페이지 내에 JAVA method를 삽입

지시어 | 지시어는 JSP에 대한 설정 정보 또는 JSP 페이지에 다른 문서를 포함 할 때 사용한다
  -<%@ page ~ %> : 페이지의 여러 설정등을 정의
  -<%@ include ~ %> : 번역 단계에서 다른 문서를 포함
  -<%@ taglib ~ %> : 태그 라이브러리 선언

내장 객체 | JSP container에 의해 명시적인 선언 없이 바로 사용 가능한 객체들. 사전에 정의 된 객체들을 뜻함
  -이번 학기 강의에서 내장 객체는 사용하지 않음.

액션 태그 | 서버나 클라이언트에게 행동을 명령하고, JSP 페이지들을 제어함.XML문법을 이용하는 것이 특징
  -jsp:include : 페이지가 요청되면, 이 태그가 위치한 곳에 파일을 포함
  -jsp:forward : 하나의 jsp페이지에서 다른 jsp페이지로 이동시킴
  -jsp:element : XML태그를 동적으로 정의
  -jsp:attritube : 동적으로 정의된 XML태그의 속성을 정의
  -jsp:body : 동적으로 정의된 XML태그의 본문을 정의
  -jsp:text : JSP페이지와 문서에 템플릿 텍스트를 작성하는데 사용
